# A Five-Year Study of File-System Metadata

[A Five-Year Study of File-System Metadata  | USENIX](https://www.usenix.org/conference/fast-07/five-year-study-file-system-metadata)

블루팀을 위한 것들이다. 
딱히 우리의 연구에서는 필요가 없을것 같다. 향후 여유가되면 살펴보겠다. 

## 서론

2000년 부터 2004년. 매년 우리 Microsoft Corporation의 Windows Desktop 컴퓨터에 있는 1만 개가 넘는 파일 시스템에서 메타데이터의 스냅샷을 수집함. 63398개의 고유한 파일 시스템으로부터 메타데이터를 수집했으며, 이중  6457개는 여러 해에 걸쳐 스냅샷을 제공했다. 

이번 연구에서는 특히 다음과 같은 요소들의 시간에 따른 변화를 분석합니다:

- 파일 크기
- 파일 나이
- 파일 유형 빈도
- 디렉터리 크기
- 파일 시스템 네임스페이스 구조
- 파일 및 디렉터리 수
- 저장 용량과 사용량
- 파일 수정 정도

기여는 총 3가지다. 

1. 수집한 데이터셋을 기여한다. 이 데이터셋은 향후 보안을 위해 정제한 후 공개. 지금까지 수집된 파일 시스템 메타데이터 중 가장 큰 규모. 가장 긴 기간을 포괄하는 데이터. 
2. 연구를 통해 얻은 모든 관찰 결과를 기여함. 내용은 아래와 같음
    - 이번 연구 기간 동안 파일 시스템에서 사용되는 공간은 증가했습니다. 이는 평균 파일 크기가 108KB에서 189KB로 증가했을 뿐만 아니라, 파일 수가 3만 개에서 9만 개로 증가했기 때문입니다.
    - 단 8개의 파일 확장자가 전체 파일 수의 35% 이상을 차지하며, 9개의 확장자가 전체 파일 용량(바이트)의 35% 이상을 차지합니다. 이 확장자들의 인기도는 수년간 지속되어 왔습니다.
    - 로컬에서 생성되거나 수정된 파일의 비율은 시간이 지남에 따라 감소했습니다. 연구 첫 해에는 파일 시스템의 중앙값 기준으로 약 30%의 파일이 로컬에서 생성되거나 수정되었지만, 4년 후에는 이 비율이 22%로 떨어졌습니다.
    - 디렉터리 크기 분포는 연구 기간 동안 큰 변화가 없었습니다. 매년 디렉터리는 하위 디렉터리를 거의 포함하지 않았고 항목 수도 적었습니다. 디렉터리의 90%는 하위 디렉터리를 2개 이하, 총 항목 수를 20개 이하만 포함하고 있었습니다.
    - 사용자 문서와 설정을 위한 네임스페이스 서브트리(subtree)에 저장된 파일 시스템 용량의 비율은 매년 증가했습니다. 처음엔 7%였지만, 마지막 해에는 15%까지 증가했습니다. 시스템 파일을 위한 서브트리에 저장된 비율도 2%에서 11%로 증가했습니다.
    - 파일 시스템의 총 용량은 연구 기간 동안 크게 증가했습니다. 중앙값 기준으로 5GB에서 40GB로 늘었습니다. 이로 인해 파일 시스템의 사용률(채움률, fullness)이 크게 줄어들 것으로 예상할 수 있지만, 실제로는 감소 폭이 크지 않았습니다. 중앙값 기준으로 채움률은 47%에서 42%로만 감소했습니다.
    - 1년 동안 80%의 파일 시스템은 더 많은 용량을 사용하게 되었고, 18%는 사용 용량이 줄어들었습니다.
3. 디렉터리 트리가 어떻게 생성되는지를 설명하는 생성적, 확률적 모델을 제안함. 
네임스페이스 트리 내에서 디렉터리가 깊이에 따라 어떻게 분포하는지를 설명할 수 있으며, 각 디렉터리당 하위 디렉터리 수의 분포 또한 설명할 수 있습니다.
